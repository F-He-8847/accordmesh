# Privacy and Security Notes

The local password derives an Argon2id key that wraps a random master key. The master key wraps independent project keys; project keys wrap independent uploaded-media keys. Versioned AES-256-GCM envelopes reject malformed nonces, truncated ciphertext, unsupported versions, and authentication failures without panicking. Sensitive in-memory keys use zeroizing buffers.

Provider credentials and sensitive Provider configuration payloads are encrypted in native storage. React receives only configured-field and masked-status metadata. The browser Mock demonstration is memory-only and cannot accept real credentials. Normal vault creation refuses to overwrite an existing vault; the separate reset command requires the literal `RESET` confirmation and removes the vault, database, projects, media, temporary files, credentials, jobs, and keys.

Uploaded files are copied to controlled temporary storage, hashed from their bytes, encrypted into managed project storage, and cleaned after processing. Real-time plaintext audio exists only during the shortest conversion window; finalized segments and Stop-time remainders are encrypted into a durable local spool for crash recovery, retry, and background finalization. Encrypted chunks are removed after the required transcript and derived outputs are durably committed. Plaintext exports use an explicit user-selected destination, remain outside the encrypted vault, and are not deleted by Reset Vault. When an external Provider is used, the audio or text required for the selected operation is sent to that Provider under its terms and retention policy.

Sensitive meeting payloads, transcripts, generated artifacts, provider credentials, managed media, and encrypted audio spool data are encrypted at rest. The current Developer Preview does not fully encrypt operational SQLite metadata. Project titles, original file names, timestamps, status values, provider/model identifiers, local storage references, and database relationships may be visible to a person or process that can read the local application database. Users should avoid sensitive project titles and file names when this metadata exposure would be a concern.

Deleting a project removes its wrapped project/media keys and managed ciphertext. Reset Vault destroys access to encrypted application data, but it does not remove plaintext files that the user previously exported to another location.
