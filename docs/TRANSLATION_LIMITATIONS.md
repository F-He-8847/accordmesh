# AccordMesh Translation Scope and Limitations

## 1. Official scope

The official first version provides:

- multilingual speech transcription when supported by the selected Provider;
- multilingual text translation;
- multilingual analysis output;
- multilingual meeting-minutes output.

It does not provide:

- translated speech playback;
- text-to-speech translation;
- simultaneous interpreted audio channels;
- guaranteed support for every language pair.

## 2. Quality limitations

Translation and transcription quality may vary because of:

- language pair;
- dialect and accent;
- code-switching;
- speaking speed;
- microphone distance;
- background noise;
- overlapping speakers;
- domain terminology;
- names and abbreviations;
- Provider and model choice.

## 3. Validation boundary

A Provider may declare support for a language that the AccordMesh maintainers have not independently validated.

Release documentation must distinguish:

- Provider-declared support;
- AccordMesh-tested language pairs;
- unvalidated community Provider claims.

## 4. Human confirmation

Users should verify critical:

- names;
- numbers;
- dates;
- amounts;
- negation;
- conditions;
- commitments;
- decisions;
- ownership and deadlines.

## 5. Source preservation

Always preserve source transcript. Display translation and core meaning as separate derived outputs.

## 6. Community extensions

Advanced speech translation, speech output, new languages, and alternative translation Providers are community extension areas. The official project supplies interfaces and contribution guidance rather than implementing every capability.
